ADBIAS

Multi-LLM 편향 조정을 통한 자동 에세이 채점 프레임워크. 4개의 LLM 채점 편향을 MFRM으로 정량화하고, Meta-LLM을 통해 편향 인지 통합 평가 수행.

2025 Team Project (5) ACL 2025 Submission

Overview

ADBIAS (Adjusting Multi-LLM Biases)는 자동 에세이 채점(AES) 시스템에서 여러 LLM의 채점 편향을 정량적으로 측정하고 보정하여 더 공정하고 신뢰성 있는 평가를 가능하게 하는 Multi-Agent 프레임워크입니다.

Problem

기존 단일 LLM 기반 AES 시스템은 모델 고유의 편향으로 인해 일관성 없는 평가 결과를 생성합니다. GPT는 중간 점수로 회귀하는 경향이 있고, Claude는 보수적으로 낮은 점수를 부여하는 경향이 있습니다.

System Architecture

ADBIAS는 3단계 파이프라인으로 구성되어 있으며, 각 단계에서 모델별 편향을 체계적으로 처리합니다.

ADBIAS 3-Stage Pipeline Architecture
1

Multi-LLM Independent Evaluation

4개의 LLM(GPT-4o, Claude 3.5, Gemini 2.5, LLaMA 4)이 동일한 에세이를 독립적으로 평가. temperature=0으로 결정적 출력 보장.

2

MFRM-based Bias Quantification

Many-Facet Rasch Model을 적용하여 각 모델의 특성별 채점 엄격도(β)를 통계적으로 측정.

3

Meta-LLM Bias-Aware Aggregation

Meta-LLM이 원본 에세이, Peer 점수/근거, 편향 메타데이터를 종합하여 최종 점수 생성.

LLM Configuration

MFRM Model

Many-Facet Rasch Model (MFRM)을 적용하여 각 모델의 채점 편향을 통계적으로 측정합니다. 교육 측정학에서 사용되는 이론적 모델을 LLM 평가에 적용한 최초의 시도입니다.

MFRM Parameter Estimation

MFRM Parameters

θ (Ability): Essay latent ability  |  β (Severity): LLM trait-specific scoring strictness  |  τ (Threshold): Score category difficulty

MFRM.py - PyTorch Implementation
class MFRM(nn.Module):
    def __init__(self, N_person, N_trait, N_rater, N_strat, K_max):
        # Essay ability (θ)
        self.person_trait = nn.Embedding(N_person * N_trait, 1)

        # Rater severity (β)
        self.rater_set_trait = nn.Embedding(N_rater * N_strat, 1)

        # Category threshold (τ)
        self.threshold_raw = nn.Embedding(N_strat, K_max)
