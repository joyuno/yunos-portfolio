Overview

AIIV는 AI 면접관과 실시간 음성 대화를 통해 면접을 연습할 수 있는 서비스입니다. 사용자의 이력서와 자기소개서를 RAG 파이프라인으로 분석하여 맞춤형 질문을 생성하고, 음성 분석을 통해 말하기 속도, 습관어, 자신감 등을 평가하여 구체적인 피드백을 제공합니다.

Key Metrics

End-to-End Latency: 1.6~2.5초 (음성 입력 → AI 응답 재생 시작)
RAG Precision: Hybrid Search + Reranking으로 40-60% 정확도 향상
Voice Analysis: 15+ 한국어 습관어 탐지, 0-100 자신감 점수

Core Features

1

Real-time Voice Interview

SSE 기반 스트리밍 파이프라인으로 실시간 음성 대화. Whisper STT → GPT-4o → OpenAI TTS 연계.

2

Document RAG Pipeline

이력서/자소서 분석을 위한 한국어 최적화 청킹, LlamaParse PDF 처리, Hybrid Search (Vector + BM25).

3

Voice Pattern Analysis

WPM 계산, 습관어(음, 어, 그러니까) 탐지, 침묵 패턴 분석, 자신감 점수화 및 개선 피드백 생성.

4

Multiple Interviewer Personas

채용담당자, 기술면접관, HR 등 다양한 면접관 캐릭터로 실전과 유사한 면접 경험 제공.

System Architecture

Streaming Pipeline

Real-time Voice Processing Flow
Audio Input → STT (Whisper) → LLM (GPT-4o) → TTS (OpenAI) → Audio Output
              ↓                ↓                ↓
           SSE Event      SSE Event        SSE Chunks
           (stt_complete) (llm_complete)   (tts_chunk)

Latency Breakdown:
- STT: 500-800ms (Whisper API)
- LLM: 800-1200ms (GPT-4o with RAG context)
- TTS First Chunk: 300-500ms (OpenAI TTS streaming)
- Total E2E: 1.6-2.5 seconds

RAG Pipeline

Document Processing & Retrieval
Document Upload Flow:
PDF → LlamaParse (complex) / Basic (simple) → Korean Chunking → Embeddings → pgvector

Query Flow:
User Question → Embedding → Hybrid Search (Vector 0.6 + BM25 0.4) → Cohere Rerank → Top K Results

Chunking Strategy (Korean Optimized):
1. Section-aware: 지원동기, 성장과정, 경력, 입사 후 포부, 장단점
2. Sentence boundary preservation
3. 800 chars/chunk with 100 char overlap
4. Token estimation (~1.5 chars/token for Korean)

Voice Analysis Metrics

Korean Document Chunking

lib/rag/chunking.ts
export function chunkKoreanDocument(text: string, options: ChunkOptions) {
  // Section-aware chunking for Korean cover letters
  const sections = ['지원동기', '성장과정', '경력', '입사 후 포부', '장단점'];

  const chunks = text
    .split(/(?=\\d+\\.\\s*(?:${sections.join('|')}))/g)
    .filter(chunk => chunk.trim().length > 50)
    .map((chunk, index) => ({
      content: chunk,
      metadata: {
        section: detectSection(chunk, sections),
        chunkIndex: index,
        tokenEstimate: Math.ceil(chunk.length / 1.5) // Korean token estimation
      }
    }));

  return chunks;
}

Voice Confidence Scoring

lib/analysis/voice.ts
export function calculateConfidence(analysis: VoiceAnalysis): number {
  const speechPace = calculatePaceScore(analysis.wpm);       // 0-100
  const fluency = calculateFluencyScore(analysis.fillerRate); // 0-100
  const articulation = calculateArticulationScore(analysis);  // 0-100

  // Weighted average
  return (speechPace * 0.3) + (fluency * 0.4) + (articulation * 0.3);
}
